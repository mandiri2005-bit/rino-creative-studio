Step 4 — Metering
+ Billing

Gerbang pre-funded credit: tiap operasi berbayar dicek & ditahan sebelum panggilan upstream, di-refund kalau gagal, di-commit sesuai biaya nyata kalau sukses. Karena COGS ~95% fee API variabel, plan flat rugi tiap user aktif — ini membuatnya mustahil secara struktural. Saat bootstrap berjalan tanpa Stripe (env-gated, dorman) dengan top-up manual.

01 Alur uang satu operasi

Setiap panggilan berbayar (chat, narasi, dst) melewati siklus ini. Saldo live ada di Redis (cache), kebenaran durable di Postgres.

1 · HOLD
Estimasi biaya → tahan kredit. Saldo kurang → HTTP 402 {needed,balance,topup_url}.
2 · CALL
Jalankan operasi upstream (Veo/Sora/chat/narasi/tts).
3a · SETTLE
Sukses → commit biaya AKTUAL, refund sisa hold, tulis usage_logs.credits.
3b · REFUND
Gagal / 0 output / cancel → kembalikan hold yang tak terpakai.

Stream yang dibatalkan menyetel porsi yang sudah diproduksi saja (sisa hold di-refund). BYOK (key sendiri) = 0 kredit, lewati hold/commit.

06 Go-live & ops

Bootstrap: METER ON, Stripe OFF

Gerbang aktif (default METERING_ENABLED=true) melindungi dari cost-bleed; Stripe dorman karena STRIPE_* kosong. Top-up = manual via CLI.

Langkah deployCatatan
Migrasi 0023+0024 → staging & mainNODE_ENV=staging node database/migrate.js lalu =production. Aditif & aman.
Merge → main → pushRailway auto-deploy. Image backend rebuild (termasuk dep stripe).
Env RailwayDefault kode sudah METER ON + $0.01 — tak wajib set. Kosongkan STRIPE_* = Stripe off.

Saat siap jualan (nyalakan Stripe)

Buat 4 produk + credit pack di Stripe Dashboard → isi STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, STRIPE_PRICE_*, STRIPE_PACK_*_CREDITS di Railway. Webhook → /webhooks/stripe. Tak perlu deploy ulang kode — hanya env.

Kill-switch

METERING_ENABLED=false mematikan gerbang total (tanpa 402, tanpa potong) — usage tetap dicatat.

Redis

Cache saldo. Outage → degradasi ke cek durable (tetap aman, tak pernah kasih spend gratis).

Idempotensi

Webhook ganda & grant --op-id sama tak pernah dobel (constraint unik di ledger + processed_stripe_events).

BYOK

User kirim X-LaoZhang-API-Key sendiri → 0 kredit; base fee tetap.

07 Definisi selesai

  • Saldo nol → operasi balas 402 + prompt top-up.
  • Saldo cukup → operasi jalan, usage_logs.credits tercatat, saldo turun sesuai katalog.
  • Gagal/cancel → kredit tak terpakai kembali.
  • Webhook ganda → saldo tidak dobel; BYOK → 0 kredit.
  • Terverifikasi: suite tests/python/test_credits_metering.py 9/9 hijau.
